# Federated Learning-Based Intrusion Detection System (IDS)

This project implements a decentralized Intrusion Detection System using **Federated Learning (FL)**. By training specialized models on distinct network attack datasets (DDoS, PortScan, and DoS) and aggregating their knowledge, the system creates a robust "Global Brain" capable of detecting multiple threat vectors without centralizing sensitive raw data.

## 🚀 Model Architectures

The system consists of three specialized client models and one global federated ensemble. All models utilize a standardized 29-feature set derived from the CIC-IDS2017 dataset.

### 1. Model 1: DDoS Specialist
* **Target Attack:** Distributed Denial of Service (DDoS) [cite: model_1.ipynb].
* **Algorithm:** Logistic Regression [cite: model_1.ipynb].
* **Key Techniques:**
    * **SMOTE Oversampling:** Used to balance the minority attack class [cite: model_1.ipynb].
    * **Log Transformation:** Applied to handle high skewness in features like `Flow Duration` [cite: model_1.ipynb].
* **Performance:** Optimized for identifying high-volume SYN flood and volumetric attacks.

### 2. Model 2: PortScan Specialist
* **Target Attack:** Port Scanning and Reconnaissance [cite: model_2.ipynb].
* **Algorithm:** Logistic Regression [cite: model_2.ipynb].
* **Key Techniques:**
    * **SMOTETomek:** A hybrid approach that oversamples the minority class and removes Tomek links to sharpen decision boundaries [cite: model_2.ipynb].
    * **Feature Pruning:** Focused on behavioral patterns like packet length variance and flag counts.
* **Performance:** High precision in identifying slow and stealthy reconnaissance scans.

### 3. Model 3: DoS Specialist
* **Target Attack:** Denial of Service (DoS) [cite: model_3.ipynb].
* **Algorithm:** Logistic Regression (Standardized for FL compatibility) [cite: model_3.ipynb].
* **Key Techniques:**
    * **Sequential Cleaning:** Custom preprocessing to handle infinite values and outliers generated by high-frequency request flows [cite: model_3.ipynb].
    * **Balanced Class Weights:** Adjusts the loss function to give equal importance to rare attack signatures.
* **Performance:** Robust detection of application-layer exhaustion attacks like Slowloris.

## 🌐 Federated Learning Implementation

### Weight Aggregation (`FL_1.py`)
The server utilizes the **FedAvg (Federated Averaging)** algorithm [cite: FL_1.py]. Key enhancements include:
* **L2 Weight Normalization:** Normalizes client coefficients to ensure one specialist doesn't dominate the global model due to feature scale differences.
* **Weighted Averaging:** Weights the contribution of each client based on their local training sample size.

### Evaluation & Testing (`FL_1_test.py`)
The global model is evaluated against unseen traffic with advanced calibration [cite: FL_1_test.py]:
* **Dynamic Thresholding:** Uses the Precision-Recall curve to find the mathematically optimal decision boundary, resolving the low precision issues observed in initial tests [cite: FL_1_test.py].
* **Robust Scaling:** Implements `RobustScaler` to ensure the model is resilient to the "spiky" nature of live network traffic [cite: FL_1_test.py].

## 🛠️ Data Preprocessing Pipeline

To ensure consistency across the federated network, all clients follow a strict pipeline:
1.  **Feature Parity:** Dropping 49 non-essential columns to maintain a 29-feature vector [cite: model_1.ipynb, model_2.ipynb, model_3.ipynb].
2.  **Infinite Value Handling:** Replacing `inf` and `NaN` with 0 [cite: model_1.ipynb, model_2.ipynb].
3.  **Outlier Clipping:** Capping extreme values at $10^{12}$ to prevent gradient explosion [cite: model_2.ipynb, model_3.ipynb].
4.  **Mathematical Transformation:** Applying `np.log1p` to "squash" wide-ranging network metrics [cite: model_1.ipynb, model_2.ipynb, model_3.ipynb].
